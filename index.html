身世浮沉雨打萍

发表于 May 25 2020   |   分类于 随笔   |  

真正感受到了人到中年的无奈和悲凉。三十岁当真是很难迈过的一个坎。

过年在家，因为疫情在家多呆了一个月，奶奶就在这段时间里永远地离开了。我爸村子里是有名的孝子，看到他伤心的样子，我也忍不住潸然泪下…

曾经看过不少人的故事，都说不能跟好朋友做同事。如今自己也算是深有体会了，我向来也不怎么在意工作里某些小成绩被人拿走，何况那也是自认为的好朋友。直到所谓的好朋友亲口说出只是拿我当做垫脚石，我彻底清醒了，我也对这个团队彻底绝望了。友情虽美，乞求不美。自己不会主动终结友情，但是话说到这个份儿上了，我还能怎么办呢？当然，不甘心也是有的，还觉着非常的失落。

在家那段时间，leader 通知我准备一下晋升 PPT，过去两年的辛苦总算可以有一些回报了。谁曾想准备了三个月，就在上周又收到消息晋升提名被取消了。感觉自己像个笑话…

媳妇公司也基本到了倒闭的边缘了，她已经三个月没拿到工资了，公司连桌子椅子都卖了，一场疫情，让他们公司像无数其他的公司一样，一下子垮掉了，而且看不到任何恢复的希望。

刚才跟爸视频通话，我问他现在怎么样了？看着躺在病床上的他，他还没开口，我再也忍不住了，瞬间泪如雨下，我妈在旁边看到我的样子，也跟着哭了起来。只有一岁多的好像永远不知道烦恼是什么的女儿在旁边哇哇乱叫。

看着女儿无忧无虑的样子，真希望她能永远都这样。而看到爸妈的样子，我更后悔自己没有早一点承担起这个家，更恨自己现在还没有能力让他们安享晚年。

在这样的三十岁之前，我曾无数次地将自己的遭遇带入到周围其他人的人生里。遇到难处了，总会想那谁不更难；春风得意了，又会想如果再好一点就可以跟那谁一样了。真正到了三十岁，才发现每个人都是不一样的，也不可能一样，莫说自己周围熟悉的人，哪怕是不认识的，也没有人会跟自己一样。

棱角再也不能那么分明了，没有哪怕一丁点任性的资本了，是时候学着做一个跟以前不一样的自己了。

在 Android Application 里 provided aar

发表于 Nov 7 2017   |   分类于 Android   |  

首先得声明一下，其实 provided aar 这样的需求场景非常少。在 com.android.library 里可以直接 provided aar，因此这样的需求只会在 com.android.application 里存在。同时还得再声明一下，这里说的 provided 一个 aar，是只引用 aar 里的类，而 aar 里的资源还是没办法引用的。

而完成插件化之后的微店，就恰恰对这样的需求非常强烈。一个 application module 里 compile 了一个aar，在其他 application moduel 里也想用这个 aar 里的类，而作者在发布 aar 的时候，并没有发布对应的 jar，这时候我们的做法往往是手动地解压出来 aar 里的 classes.jar，然后再在 dependencies 里添加这个 classes.jar 的 provided 依赖。看上去很简单嘛，可问题是遇到一个就要这样做一次，这就好麻烦了。那么有没有一种纯自动的方式去做这个事情呢？这篇文章讲的就是这个事情。
